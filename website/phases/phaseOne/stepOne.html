Minimorphic / SLATE / Phase One

Reality Check (Prototype Brain)

I am not making a CPU. I am not taping out silicon. I am not becoming Intel in a garage.

I am building an object that contains a computer.

That means I pick a brain that already exists, that has documentation, has drivers, has a community, and boots without summoning god knows what.

What I actually want

Laptop prototype requirements, in plain language:

  • Enough power to feel like a laptop (not a toy)
  • Low power draw so I can run it off a real battery
  • Real OS support (Linux is the minimum; Windows ideally.)
  • Affordable hardware I can buy today
  • Documentation that doesn’t require NDAs and prayer

Three sane paths

Path 1: Raspberry Pi Compute Module (ARM)

This is the “I actually want to finish this project” option.

  • Huge documentation and community
  • Easy power requirements
  • Small footprint
  • Can later design a carrier board as a real Minimorphic step

Tradeoff: not ultrabook-fast. But it’s perfect for proving the object and the module system.

Path 2: Intel N100 / N95 class (x86)

This is the “real laptop feel without enterprise complexity” typeshit option.

  • x86 compatibility: things behave like a normal PC
  • Low power, workable thermals
  • Good performance per watt for early prototypes
  • Dev boards exist at reasonable prices

Tradeoff: less open ecosystem, and custom board design later gets more painful. Like no doc. So probably not.

Path 3: AMD Ryzen embedded / laptop-class modules

This is the “oh for fucks sake” option.

  • Serious performance
  • Real ultrabook class

Tradeoff: cost and complexity. Documentation can be restricted. Power and routing get real, fast.

The point you can’t skip

Not exactly choosing the final CPU right now.

Simply are choosing the development brain that lets me design the object.

The win in Phase One is not benchmarks. The win is a working Minimorphic object: enclosure, modules, power plan, thermals, and a clean integration story.

Pick: ARM (easier, documented) or x86 (more “laptop”) Then: build the object around it Then: iterate toward custom hardware

Phase One outcome: define the object before building the parts.
